Day 7

Monitor와 analysis port

monitor가 DUT signal을 관찰해서 transaction으로 복원하고 scoreboard/coverage로 broadcast하는 구조를 이해한다.

오늘의 완료 기준monitor output이 scoreboard와 coverage로 흘러가는 경로를 설명한다.
학습 루프개념 -> 실제 파일 -> 흐름 추적 -> 퀴즈 -> 작은 실습 순서로 진행합니다.
이전 Day다음 Day전체 목차

mental model

오늘 잡을 개념

monitorDUT를 drive하지 않고 signal을 transaction으로 복원한다.
analysis_port한 producer가 여러 subscriber에게 transaction을 broadcast한다.
observer patternmonitor는 scoreboard 내부를 몰라도 write로 전달한다.

trace

읽는 순서

  1. CYC/STB 조건 확인
  2. transaction field 복원 확인
  3. analysis_port.write와 env connect 확인
