Day 1

UVM 전체 구조 mental model

test, env, agent, sequencer, driver, monitor, scoreboard, coverage, transaction이 왜 분리되는지 데이터 흐름으로 이해한다.

오늘의 완료 기준transaction -> sequence -> sequencer -> driver -> interface -> DUT -> monitor -> scoreboard/coverage 흐름을 내 말로 설명한다.
학습 루프개념 -> 실제 파일 -> 흐름 추적 -> 퀴즈 -> 작은 실습 순서로 진행합니다.

mental model

오늘 잡을 개념

역할 분리UVM은 의도, 구동, 관찰, 비교, coverage를 분리한다.
transaction 중심pin 하나가 아니라 bus operation object가 흐른다.
env 조립agent, scoreboard, coverage를 조립해 검증환경을 완성한다.
