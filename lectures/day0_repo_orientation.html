mental model

UVM testbench를 데이터 흐름으로 읽기

처음에는 class 이름보다 흐름이 중요합니다. UVM은 추상적인 transaction을 만들고, driver가 pin-level signal로 바꾸고, monitor가 다시 transaction으로 관찰해서 scoreboard와 coverage로 보냅니다.

entry
tests/wb_env_tb_mod.sv -> run_test()
test
tests/wb_env_test.sv -> env 생성, config_db 설정, default_sequence 선택
stimulus
src/wb_transaction.sv -> src/wb_master_agent_sequence_library.sv
drive
src/wb_master_seqr.sv -> src/wb_master.sv -> src/wb_master_if.sv
observe
src/wb_master_mon.sv, src/wb_slave_mon.sv -> analysis_port
judge
src/wb_scoreboard.sv, src/wb_env_cov.sv
한 문장으로 요약

sequence가 검증 의도를 만들고, driver가 의도를 핀으로 바꾸며, monitor가 핀 변화를 관찰하고, scoreboard와 coverage가 결과와 충분성을 판단합니다.

아래 발췌들은 위 data flow 각 줄의 실제 코드입니다. Day 0에서는 전체를 완벽히 이해하려고 하지 말고, “이 파일이 어떤 역할을 맡는가”만 표시하면서 읽으면 됩니다.
