Day 6

Interface와 Driver

transaction이 실제 DUT pin-level signal로 변환되는 과정을 이해한다.

오늘의 완료 기준transaction 하나가 driver를 거쳐 Wishbone signal로 바뀌는 과정을 설명한다.
학습 루프개념 -> 실제 파일 -> 흐름 추적 -> 퀴즈 -> 작은 실습 순서로 진행합니다.
이전 Day다음 Day전체 목차

mental model

오늘 잡을 개념

interfacebus signal bundle과 clocking block을 묶는다.
drivertransaction을 pin-level signal로 drive한다.
timingclocking block과 wait loop가 cycle 동작을 만든다.

trace

읽는 순서

  1. interface signal 확인
  2. seq_item_port handshake 확인
  3. read/write task의 field-to-signal 매핑 확인
