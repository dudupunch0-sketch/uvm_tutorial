29.07.2002

22.07.2002
~~~~~~~~~~
이 파일이 한동안 갱신되지 않았으므로 모든 변경 사항이 포함된 것은 아닙니다.

Bug Fixes:
 * 느린 baud rate에서 동기화가 손실되거나 stop bit가 잘못 수신될 수 있는 문제가 수정되었습니다.
  문제는 Kenny.Tung이 보고했습니다.
 * 잘못되었거나 누락된 loopback 처리가 수정되었습니다. Cherry Withers가 보고했습니다.

Improvements:
 * 가능한 곳에서는 FIFO를 일반적인 inferred memory로 만들었습니다.
 따라서 FPGA에서는 RAM(Xilinx의 Distributed RAM)으로 infer될 것입니다.
 이것은 Slice count를 약 1/3 절감하고 P&R 및 synthesis 시간을 줄입니다.

 * 선택적 baudrate output(baud_o)을 추가했습니다.
 이것은 16550 chip의 BAUDOUT* signal과 동일합니다.
 16xbit_clock_rate, 즉 분주된 clock을 출력합니다.
 기본적으로 비활성화되어 있습니다. 사용하려면 UART_HAS_BAUDRATE_OUTPUT을 define하십시오.

Note:
 uart_defines.v 파일은 더 이상 source files에 포함되지 않습니다.
 따라서 simulation을 수행할 때 이 점을 염두에 두십시오. 수동으로 추가하십시오.
 각기 다른 configuration을 위해 자체 define file을 사용할 수 있도록 이렇게 했습니다.
 제가 개발 중인 IrDA core에 이 기능이 필요합니다.
 이전 동작을 원한다면 `includes의 comment만 해제하면 됩니다.
 uart_fifo.v 파일은 더 이상 사용되지 않습니다. 대신 uart_rfifo.v와 uart_tfifo.v
 파일이 이제 있습니다. 또한 raminfr.v는 새로운 inferred ram module입니다.

 새 core를 확인해 보시고 마음에 드시길 바랍니다.

23.06.2001
~~~~~~~~~~

* Bob Kirstein의 도움으로 두 개의 bug가 더 수정되었습니다:
   1. Transmitter가 stop bit를 필요한 것보다 두 16xclock cycle 더 길게 전송하고 있었습니다.
   2. Receiver가 각 character마다 1 16xclock cycle을 잃어 동기화가 어긋났습니다.

* 주요 변경 사항:
    고속 system에 필요하다고 생각했던 32-bit 대신 divisor latch register를 16-bit 길이로
    수정했습니다. 이 점을 지적해 준 Rick Wright에게 감사합니다.
    따라서 이제 DL3와 DL4 register byte는 사용되지 않습니다.
    Documentation은 이 변경 사항을 따르도록 갱신되었습니다.

* 한 byte에서 1개를 초과하는 stop bit는 구현되어 있지 않습니다.

2.05.2001
~~~~~~~~~

* transmitter와 receiver를 수정했습니다. start bit와 stop bit가 반전되어 송수신되고 있었습니다.
  이 점을 지적해 준 Bob Kirstein에게 큰 감사를 전합니다.


31.05.2001
~~~~~~~~~~

* register read code에서 사소한 변경을 했습니다
* FCR을 2 bits wide로 변경하고(reset bit는 필요하지 않음), 대신 제가 구현하는 것을 잊었던 rx_reset 및 tx_reset
   signal을 활성화했습니다.
* FCR에 대한 define을 변경했습니다.
* top-level에서 연결되지 않은 port를 정리했습니다.
* 추가 gate count 비용을 감수하고 versioning 문제를 해결하기 위해 code를 두 개가 아닌 하나의 FIFO module만
   갖도록 변경했습니다. UART_RX_FIFO는 약간 수정되어 UART_FIFO로 이름이 변경되었습니다.
* UART_RX_FIFO.v 및 UART_TX_FIFO.v 파일을 project에서 제거했습니다.
* FIFO handling과 관련하여 receiver 및 transmitter module을 변경했습니다.
* UART_top.v와 test bench를 제외한 모든 파일에서 `include "UART_defines"를 comment 처리했습니다.
* 조금 더 나은 check를 위해 test bench를 약간 수정했습니다.


29.05.2001
~~~~~~~~~~

* 수정: Line Control Register block의 sensitivity list에 wb_rst_i가 없었습니다
* 수정: Modem Status Register block의 sensitivity list에 wb_rst_i가 없었고 reset value를 설정하지 않았습니다

27.05.2001
~~~~~~~~~~

많은 bug를 지적해 준 Rick Wright에게 감사합니다.

* rf_pop 및 lsr_mask flag가 deassert되지 않는 문제를 수정했습니다.
* Time-Out interrupt가 IER의 bit 0에 의해 mask되지 않는 문제를 수정했습니다
* interrupt logic이 IER에 의해 mask되지 않는 문제를 수정했습니다
* IIR의 bit 0(interrupt pending)이 잘못 설정되는 문제를 수정했습니다
* Modem Status Register bits 3:0 handling을 수정했습니다(정상적으로 동작하지 않았습니다)
* modem status interrupt가 MSR의 bits 7:4 대신 bits [3:0](deltas)와 관련되도록 수정했습니다.
   이렇게 하면 MSR을 읽을 때 interrupt가 clear됩니다.
* IIR을 읽어도 THRE interrupt가 reset되지 않는 문제를 수정했습니다
* Receiver 및 Transmitter FIFO가 #include command 문제 때문에 FIFO_inc.v 파일을 사용하지 않도록 변경했습니다.
* CVS tree에서 FIFO_inc.v를 제거했습니다.

* specifications .pdf 파일을 갱신했습니다
